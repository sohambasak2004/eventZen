Architecture Issue Explanation

Current System:

  • ✅ Events: Stored in localStorage (frontend only)
  • ✅ Venues: Stored in MongoDB (backend service)
  • ❌ Event-Venue Bookings: Reference eventIds that don't exist in backend

The Problem:

  • Events are deleted from localStorage
  • But related booking records in MongoDB still reference the deleted eventIds
  • This creates orphaned data in the database
